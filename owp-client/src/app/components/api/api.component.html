• {{api.name}}

    • authentication

      Для доступу до більшості апі потрібно бути авторизованим користувачем, в нашому випадку, мати в хедері валідний токен типу Беарер при запиті до роутів.

      POST /api/auth/register - приймає емайл-адресу та рядок що містить пароль майбутнього користувача, якшо такий емейл уже зареєстрований - запит повертає помилку з кодом 409 - конфлікт. В іншому випадку ми хешуємо пароль і зберігаємо нашого користувача до бази даних.

      POST /api/auth/login - приймає емайл-адресу та рядок з паролем існуючого користувача. Якщо паролі не співпадають - повертається статус 401 - неавторизований, у випадку якзо не співпадає адреса - 404 (не знайдено). Якщо дані було введено коректно - повертається JWT токен який надає змогу користувачу користуватися сайтом.

    • user

      Вміст даної таби навчить вас користуватися АПІ для взаємодії з користувачами

      GET /api/auth/:role - повеертає масив юзерів з вказаною роллю

      GET /api/auth/user/:id - повертає користувача по заданому ід

      DELETE /api/auth/:id - видаляє користувача по заданому ід

      PATCH /api/auth/:id - апдейтить користувача по заданому ід

    • category

      Тут буде описано які запити можна використовувати для взаємодії з композиційними сутностями

      GET /api/category/page/:page - запит використовується для того щоб отримати масив об'єктів фіксованого розміру. Він приймає номер сторінки і дістає із загальної маси необхідний відрізок пропустивши (limit * page - limit) елементів.

      GET /api/category/:id - повертає категорію по заданому ід

      POST /api/category/ - створення нової сутності, приймає ім'я та картинку, обробляючи форм дату npm пакетом multer і відправляючи її на сервер для зберігання медіафайлів - cloudinary.

      PATCH /api/category/:id - апдейтить категорію по заданому ід

      DELETE /api/category/:id - видаляє категорію по заданому ід

    • order

      Набір запитів, який скоріше за все ще поповниться, який допоможе вам зручно працювати з замовленнями.

      GET /api/order/ - повертає всі замовлення колекції

      POST /api/order/ - створює нове замовлення

      GET /api/order/:userId - повертає всі замовлення які належать конкретному користувачу

    • position

      GET /api/position/:categoryId - приймає ІД категорії, позиції якої ми хочемо отримати, та строка з фільтром який будуть містити імена всіх позицій. Повертає список всіх отриманих сутностей.

      POST /api/position/ - створює нову сутність

      PATCH /api/position/:id - модифікує позицію по заданому ІД

      DELETE /api/position/:id - видаляє позицію по заданому ІД

    • table

      Колекція запитів, що навчать вас створювати, видаляти, апдейтити та отримувати необхідні столики.

      GET /api/table/ - повертає список всіх столиків

      GET /api/table/:id - повертає стіл по заданому ІД

      POST /api/table/ - створює новий столик, прийнявши кількісць місць за ним

      PATCH /api/table/:id - модифікує стіл по заданому ІД

      DELETE /api/table/:id - видаляє позицію по заданому ІД

    • errors

      Більшість запитів загорнуті у try[ ... ] catch (error) [ ... ], який повертає статус 500 ( внутрішня помилка сервера ) і json який дає вам змогу дізнатися про її походження.

      401 - неавторизований, з'являється коли користувач намагається зробити запит без наявності валідного Беарер токена у відповідному хедері.
      404 - не знайдено, статус повертається коли користувач задає не валідний ІД користувача, Категорії, або намагається перейти до невідомої сторінки.
      409 - конфлікт - виникає коли користувач хоче зробити декілька акаунтів, використовуючи один і той самий емайл.
